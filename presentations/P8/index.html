Application Programming Interface (API)

Allows one piece of software (in our case web application) to talk to another.

APIs are everywhere: software libraries, OS APIs, remote APIs, smartphone APIs, audio/video APIs, web APIs etc.

Different types of web APIs exist but most common is a REST API

  • REpresentational State Transfer (REST)
  • Works mostly like a website over HTTP protocol
  • Stateless, Cacheable, Client-Server
  • When you make an API request, you (most frequently) get the response back in JSON

JSON - What is it?

  • JSON is short for JavaScript Object Notation
  • It is lightweight text-based open standard data-interchange format
    • a simple, text-based way to store and transmit structured data
    • it provides a human-readable collection of data that can be accessed in a logical manner.
  • It was popularized by Douglas Crockford (2006).
  • A JSON object can be stored in its own file, which is basically just a text file with an extension of .json, and a media (MIME) type of application/json.
  • JSON has taken over the world!

JSON Syntax

  • JSON syntax is derived from JavaScript object notation syntax:
    • Data is in name/value pairs
      • A name/value pair consists of a field name (key) in double quotes, followed by a : followed by a value.
      • Example: { "name":"Mitsos" }
    • Data is separated by commas
      • Example: { "model":"Mustang" , "color":"Red", "year":2010 }
    • Curly braces hold objects
      • Example: { car: { "model":"Mustang" , "color":"Red", "year":2010 } }
    • Square brackets hold arrays
      • Example: { "students":[ "Mitsos", "Kitsos", "Litsos"] }

JSON Syntax

Number integer or floating point, e.g. { “age”:30 } or { “salary”:2340.68 }
String in double quotes, e.g. { “firstname”:“mitsos” }
Boolean true or false, e.g. { “pass”:false }
Array collection of values, e.g. { “employees”: [“Mitsos”,“Kitsos”,“Ritsos”] }
Object e.g. { "employee": { "id":5, "name":"Mitsos", "position":"contract killer" } }
Null e.g. { "phonenumber":null }

JSON - Some Notes

  • JSON is purely a data format — it contains only properties, no methods.
  • JSON requires double quotes to be used around strings and property names.
  • Even a single misplaced comma or colon can cause a JSON file to go wrong, and not work. You should validate any data you are attempting to use (computer-generated JSON is less likely to include errors).
  • JSON can actually take the form of any data type that is valid for inclusion inside JSON, not just arrays or objects. For example, a single string or number would be a valid JSON object.

JSON - Accessing the data

JSON and JavaScript - parse()

  • Your web app may request and receive JSON data from a server
    • This data will (most likely) be returned as a STRING
  • We will need to parse the data so that it becomes a JavaScript object.
    • To do this, we use the parse() function

JSON and JavaScript - stringify()

  • Your web app may need to send JSON data to a server
    • Data can only be send as text (a string)
  • You will need to convert a JavaScript object into a string
    • To do this, we use the stringify() function

JavaScript Callback Functions

  • A callback is a function that is to be executed after another function has finished executing — hence the name 'call back'.
  • In JavaScript, functions are objects. Because of this:
    • functions can take functions as arguments
    • and can be returned by other functions
  • Functions that do this are called higher-order functions.
  • Any function that is passed as an argument is called a callback function.

AJAX - What is it?

  • AJAX stands for Asynchronous JavaScript And XML
  • AJAX is a technique to send/receive data to/from server asynchronously without reloading the whole web page.
  • It is a group of inter-related technologies like JavaScript, DOM, XML, HTML/XHTML, CSS, XMLHttpRequest etc.
  • An XMLHttpRequest object is used for asynchronous communication between client and server: it performs the operations:
    • Sends data from the client in the background
    • Receives the data from the server
    • Updates the web page without reloading it
  • Where is AJAX used? EVERYWHERE!

How Ajax Works

  1. User sends a request from the UI and a JavaScript call goes to XMLHttpRequest object.
  2. HTTP Request is sent to the server by XMLHttpRequest object.
  3. Server interacts with the database (using Node, PHP, Python, Ruby, .NET etc.)
  4. Data is retrieved.
  5. Server sends data (JSON,XML) to the XMLHttpRequest callback function.
  6. HTML and CSS data is displayed on the browser.
